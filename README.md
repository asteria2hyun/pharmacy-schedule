# 근무약사 스케줄 관리 MVP

모바일 브라우저에서 바로 열어볼 수 있는 정적 웹앱입니다. 현재 버전은 서버 없이 브라우저 저장공간에 데이터를 저장합니다.

## 실행

`index.html` 파일을 브라우저에서 열면 됩니다.

## 계정

공개 로그인 화면에는 계정 정보가 표시되지 않습니다.

신규 등록 계정은 초기 비밀번호가 `1111`입니다. 첫 로그인 때 새 비밀번호를 설정해야 앱에 들어갈 수 있습니다. 관리자는 직원관리에서 각 계정의 비밀번호를 다시 `1111`로 초기화할 수 있습니다.

## 정적 배포

`index.html`, `app.js`, `styles.css` 세 파일을 같은 폴더에 올리면 정적 호스팅에서 열 수 있습니다.

주의: 현재 정적 버전은 서버 데이터베이스가 없으므로 기기별 브라우저 저장공간에 데이터가 따로 저장됩니다. 여러 사람이 같은 데이터를 실시간 공유하는 운영 버전은 Supabase 같은 백엔드 연결이 필요합니다.

## 주요 범위

- 아이디/비밀번호 로그인, 퇴사자 접근 차단
- 초기 비밀번호 `1111`, 첫 로그인 비밀번호 설정, 관리자 비밀번호 초기화
- 모바일 하단 탭 UI
- 월간 근무표, 한국 기준 토요일/일요일/공휴일 색상 표시
- `10-10` 근무자 우선 표시, 그 아래 `10-8` 표시
- 직원 라인 별도 표시 및 구분값으로 관리자/근무약사/직원1/직원2 관리
- 근무 변경 요청/승인/거절/관리자 취소
- 교체 가능 조합: 관리자 ↔ 근무약사, 근무약사끼리, 직원1 ↔ 관리자
- 직원2는 일반 교체 없이 근무변경 탭에서 관리자 대체 근무만 요청
- 관리자 대체 승인 시 배주성이 직원포지션에 배치될 수 있으며, 같은 날짜 약사포지션과 직원포지션 중복 배정도 허용
- 관리자 근무 변경 관리 탭에서 근무약사 변경완료 내역을 최근 처리순으로 확인하며 지난 근무 완료건은 목록에서 제외
- 직원1 연차 사용은 관리자 승인 후 처리완료
- 교체는 근무시간을 바꾸지 않고 사람만 교체
- 2026년 5월, 7월은 사진 기준 실제 근무표로 세팅
- 2026년 8월부터는 요일별 기본 패턴으로 월별 근무표 자동 초기세팅
- 교체 승인 후 약사/관리자 근무와 직원 근무를 함께 반영해 급여 시간 계산
- 관리자 근무표 수정에서 `10-10`, `10-8`, `직원(10-8:30)` 근무를 추가/수정/삭제
- 관리자 근무표 수정에서 날짜 입력으로 해당 날짜 근무자를 확인하고, 시작/종료 시간을 바꿔 급여 시간을 재계산
- 개인 급여, 관리자 급여관리, 직원관리, 연차관리
- 직원별 급여 적용 시작일, 마지막 근무일, 요일별 근무 변경 시작일 관리
- 평일 시급과 주말/공휴일 시급 기준 급여 계산

## 데이터 구조

- `employees`: 이름, 로그인, 비밀번호 변경 필요 여부, 구분, 재직 상태, 마지막 근무일, 급여 기준, 급여 적용일, 연차
- `schedules`: 약사/관리자 근무 날짜, 근무 종류, 근무자
- `staffSchedules`: 직원 근무 날짜, 근무자, 급여 계산용 시간
- `swapRequests`: 요청자/상대방, 양쪽 근무 항목, 상태, 처리 시간
- `holidays`: 한국 공휴일 및 추후 API 연동용 출처 정보
- `auditLogs`: 관리자 변경 및 교체 기록

실제 운영 배포 버전에서는 Supabase Auth, PostgreSQL, Row Level Security로 급여와 근무 변경 요청 접근 권한을 서버에서 강제하는 구성이 적합합니다.
